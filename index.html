Setup

Getting it running

Copy the commands. They work.

1

Clone the design kit — once

Terminal 
mkdir -p ~/zendesk/design-kits ~/zendesk/prototypes
cd ~/zendesk/design-kits
git clone git@github.com:zendesk/employee-service-design-kit.git

You only do this once.

2

Create project folder

Terminal 
mkdir ~/zendesk/prototypes/my-new-project

Name it whatever you want.

3

Open in Cursor

File → Open Folder, navigate to your project folder, click Open.

4

Open terminal

Press Ctrl + ` or Terminal → New Terminal

5

Run setup script

Terminal 
~/zendesk/design-kits/employee-service-design-kit/scripts/setup-here.sh itam-shell

Copies template, installs dependencies, sets up git. Takes ~30 seconds.

6

Start dev server

Terminal 
npm run dev

Open http://localhost:5173 in your browser.

✓

Done

Your prototype is running. Start building in <Main>.

Workflow

From local to production

When you're ready to share, version, and hand off to developers.

✨ New: Simplified Commands (July 2026)

The template now includes wrapper scripts for all workflow commands. Run ./publish.sh, ./check-compliance.sh, ./check-translations.sh, etc. directly from your project folder — no more typing long paths! Scripts automatically handle JFrog authentication and find the design kit installation.

1

Keep building locally

You're already here

No scripts needed. Just code.

2

Publish to GitHub

When ready to share

Terminal 
./publish.sh

Creates private repo + live GitHub Pages site. Automatically configures npm authentication for Zendesk packages.

3

Push updates

When you make changes

Terminal 
./push-update.sh

Version bumping + CHANGELOG + site rebuild.

4

Production checks

Before developer handoff

Code 
./check-compliance.sh
i18n 
./check-translations.sh
a11y 
./audit-accessibility.sh
5

Finalize handoff

When completely done

Terminal 
./finalize-handoff.sh

Runs all checks, generates docs, bumps to v1.0.0.
