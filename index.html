The idea

The shell is built. You build the content.

The template ships with Zendesk's header, product tray, and sidebar. Your prototype goes inside the <Main> area.

Included in the template

  • Full header with product tray

What you build

  • Your page content
  • Tables, forms, and lists
  • Custom components
  • Data and logic
  • Everything inside <Main>
Key conceptthe template provides the shell (navigation); you build the content (your pages), all inside <Main>.
Quick start · Cursor workflow

The six steps

From zero to a running prototype. Copy each command with one click.

1

Clone the design kit — one time only

zsh
mkdir -p ~/zendesk/design-kits ~/zendesk/prototypes
cd ~/zendesk/design-kits
git clone git@github.com:zendesk/employee-service-design-kit.git

You only do this once — the design kit stays there.

2

Create your project folder

zsh
mkdir ~/zendesk/prototypes/my-new-project

Use descriptive names like asset-fields-v2 or status-redesign.

3

Open the folder in Cursor

  • Click File › Open Folder
  • Navigate to ~/zendesk/prototypes/my-new-project
  • Click Open
4

Open the integrated terminal

Press Ctrl ` or use Terminal › New Terminal.

5

Run the setup script

zsh
~/zendesk/design-kits/employee-service-design-kit/scripts/setup-here.sh itam-shell

Copies the template, installs dependencies, and initializes git. ~30 seconds.

6

Start the dev server

zsh
npm run dev

Open http://localhost:5173 in your browser.

Doneyour prototype is running with full Zendesk navigation and Flora styling.

Code example

App.tsx

  <div style={{ padding: '20px 40px' }}>
    <h1>My page title</h1>

      {/* Your content here */}
    </div>
  </div>
</Main>
Admin Center ↔ Support

The product switcher

The template includes both products. Users switch between them with the product tray.

  • Click the product tray button (top-left, reads "Admin center")
  • Select Support from the dropdown
  • The page switches to the Support view with its own navigation
  • Click again to switch back
Tipeach product has its own page file: src/pages/FieldOrder.tsx (Admin Center) and src/pages/SupportFieldOrder.tsx (Support).

Only need one product? Disable the switcher.

In src/App.tsx, drop the products prop on line 83:

App.tsx
// Before
<Product key={currentProduct} locale="en-US" isFlora products={products}>

// After (no products prop = no product tray)
<Product key={currentProduct} locale="en-US" isFlora>

In src/App.tsx, remove the switcher logic and render one view:

App.tsx
// Remove useState and product switching.
<Product locale="en-US" isFlora>
  <AppHeader product="admin-center" />
  <Sidebar />
  <Main id="main-content">
    <FieldOrder />
  </Main>
</Product>
Stay in your lane

Change vs. keep

A simple rule keeps your prototype upgrade-safe and on-brand.

Change these freely

  • src/pages/
  • src/components/content/
  • src/data/
  • Content inside <Main>

Leave these alone

  • src/components/shared/
  • Header.tsx
  • Sidebar.tsx
  • <Product> shell in App.tsx
