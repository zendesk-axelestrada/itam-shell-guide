Get started

The 6 steps

From zero to a running prototype. Copy each command with one click.

1

Clone the design kit — one time only

Terminal 
mkdir -p ~/zendesk/design-kits ~/zendesk/prototypes
cd ~/zendesk/design-kits
git clone git@github.com:zendesk/employee-service-design-kit.git

You only do this once. The design kit stays there forever.

2

Create your project folder

Terminal 
mkdir ~/zendesk/prototypes/my-new-project

Use descriptive names like asset-fields-v2 or status-redesign.

3

Open the folder in Cursor

In Cursor:

  • Click File → Open Folder
  • Navigate to ~/zendesk/prototypes/my-new-project
  • Click Open
4

Open the terminal

Press Ctrl + ` or click Terminal → New Terminal

5

Run the setup script

Terminal 
~/zendesk/design-kits/employee-service-design-kit/scripts/setup-here.sh itam-shell

This copies the template, installs dependencies, and sets up git. Takes about 30 seconds.

6

Start the dev server

Terminal 
npm run dev

Open http://localhost:5173 in your browser.

🎉

Done! Your prototype is running with full Zendesk navigation.

Beyond local dev

From local to production

Built something great? Here's how to share it, version it, and hand it off to developers—all automated.

New

These scripts were added in June 2026 based on the column-visibility prototype workflow.

1

You're already here

Keep building locally until you're ready to share

The 6 steps above got you to a working prototype. Keep iterating, testing, and building. No scripts needed—just code!

2

Share it with your team

When: Ready to show stakeholders

Terminal 
~/zendesk/design-kits/employee-service-design-kit/scripts/publish-to-zendesk.sh

Creates a private repo at zendesk/itam-{your-project}-prototype + a live GitHub Pages site.

3

Push updates

When: Made changes, want to update the live site

Terminal 
~/zendesk/design-kits/employee-service-design-kit/scripts/push-update.sh

Handles version bumping (patch/minor/major) + CHANGELOG updates + site rebuild. All automatic.

4

Run production checks

Optional, but helpful before developer handoff

Three scripts to check quality:

Code quality 
~/zendesk/design-kits/.../scripts/check-production-compliance.sh
Translations 
~/zendesk/design-kits/.../scripts/check-translations.sh
Accessibility 
~/zendesk/design-kits/.../scripts/audit-accessibility.sh
5

Hand it off to devs

When: 100% done and ready for production

Terminal 
~/zendesk/design-kits/employee-service-design-kit/scripts/finalize-handoff.sh

Runs all checks, generates IMPLEMENTATION.md, bumps to v1.0.0, and creates a complete handoff package.

Documentation

Full docs in scripts/README.md — examples, troubleshooting, and detailed guides for each script.
